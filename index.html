Proyecto Destacado

Pantalla Volumetrica para Simulaciones Quirurgicas de Columna Lumbar

1er puesto - Facultad 2do puesto - General S/. 5,000.00 Articulo IEEE

Diseno de los lineamientos de una pantalla volumetrica de barrido traslacional vertical para simulaciones quirurgicas de columna lumbar. El proyecto aborda la problematica de la visualizacion 3D en cirugia espinal, proponiendo un sistema que proyecta multiples cortes bidimensionales (slices) de imagenes medicas DICOM sobre una pantalla translucida movil, generando volumenes 3D reales mediante persistencia retiniana. Se desarrollo una Matriz de Trazabilidad de Requisitos (RTM) vinculando necesidades medicas con metricas de rendimiento. El sistema incluye procesamiento de imagenes DICOM, reconstruccion 3D con Marching Cubes, voxelizacion, y un modelo matematico de barrido vertical sincronizado.

Python DICOM Marching Cubes Procesamiento 3D Unidades Hounsfield Proyeccion DLP IEEE

Python Scapy SMTP TCP/IP Sockets

Bitácora del Curso

Bitácora de Clases y Aprendizajes Semanales — Desarrollo de Aplicaciones Web (Fase I)

Semana 1: Arquitectura y Evolución Web

Teoría y Modelamiento HTTP / REST / SOAP

¿Qué se realizó? Análisis evolutivo desde la Web 1.0 estática hasta las arquitecturas modernas basadas en microservicios. Evaluamos a fondo el flujo de comunicación Cliente-Servidor y el ciclo de vida de las peticiones HTTP/HTTPS, mapeando las diferencias estructurales entre servicios SOAP y arquitecturas RESTful.

Lo que entendí: Comprendí que el backend debe estar completamente desacoplado del frontend mediante APIs bien definidas. Diseñar pensando en el protocolo HTTP y el manejo correcto de sus estados (200, 404, 500) es la base para lograr interoperabilidad, escalabilidad y un procesamiento eficiente en el servidor.
Cliente-Servidor HTTP Status API REST

Semana 2: Ingeniería de Requerimientos Web

Gestión de Proyectos Stakeholders

¿Qué se realizó? Captura, clasificación y especificación de requerimientos funcionales (RF) y no funcionales (RNF) orientados a plataformas web. Analizamos métricas críticas de rendimiento para la infraestructura, disponibilidad del sistema, concurrencia de usuarios en tiempo real y políticas esenciales de seguridad.

Lo que entendí: En desarrollo web, los RNF son tan cruciales como los funcionales. Una aplicación puede hacer sus tareas de cálculo, pero si no soporta 500 peticiones concurrentes o tarda más de 2 segundos en responder, el sistema fracasa. Aprendí a documentar técnicamente antes de codificar.
RF / RNF Concurrencia Métricas web

Semana 3: Diseño de Arquitectura y Patrones

Patrones de Software Desacoplamiento

¿Qué se realizó? Estudio profundo de patrones arquitectónicos para la web, centrándonos en el modelo de capas y el patrón Modelo-Vista-Controlador (MVC). Analizamos cómo este enfoque facilita la separación de responsabilidades entre la lógica de negocios, el almacenamiento de datos y la interfaz de usuario.

Lo que entendí: Entendí que la arquitectura limpia previene el código espagueti. Utilizar capas o MVC permite modificar la interfaz visual o migrar un motor de base de datos de manera independiente sin alterar las reglas de negocio del sistema, optimizando costos de mantenimiento futuro.
Patrón MVC Clean Architecture Modularidad

Semana 4: Modelamiento Funcional del Sistema

Diagramas UML Flujos de Usuario

¿Qué se realizó? Modelamiento avanzado mediante especificaciones formales de casos de uso y diagramas de secuencia aplicados a entornos web. Mapeamos flujos complejos de autenticación, sesiones de usuario e interacciones dinámicas con componentes externos de software.

Lo que entendí: Los casos de uso web deben contemplar el comportamiento asíncrono y las posibles interrupciones de conexión. Modelar los diagramas de secuencia me enseñó a prever excepciones de flujo y errores antes de la fase de construcción, reduciendo fallas lógicas.
Casos de Uso Secuencia UML Lógica de Flujo

Semana 5: Prototipado e Ingeniería UI/UX

Wireframing Usabilidad

¿Qué se realizó? Diseño de maquetas estructurales (wireframes) y prototipos dinámicos de alta fidelidad. Estudiamos las heurísticas de Jakob Nielsen, principios fundamentales de accesibilidad web (WCAG) y técnicas de diseño adaptativo (Responsive Design) para móviles y escritorio.

Lo que entendí: La experiencia de usuario es ciencia aplicada. Entendí que un sistema web puede ser robusto por detrás, pero si su interfaz confunde al cliente o no se adapta a pantallas móviles, pierde toda utilidad. El diseño UI/UX estructura la información para mitigar la carga cognitiva del usuario.
Heurísticas Nielsen Responsive UI WCAG

Semana 6: Modelamiento de Datos y Persistencia

Diseño de BD Integridad Referencial

¿Qué se realizó? Diseño lógico y conceptual de bases de datos relacionales integradas al ecosistema web. Trabajamos en la normalización de tablas (1FN, 2FN, 3FN), optimización de índices para consultas concurrentes rápidas y definición precisa de restricciones de integridad referencial.

Lo que entendí: La persistencia de datos es el corazón de la integridad empresarial. Aprendí que una base de datos mal normalizada genera redundancia, lentitud en consultas web críticas y corrupción de estados. Diseñar relaciones sólidas garantiza la consistencia de los datos transaccionales.
Normalización SQL Server / SQLite Performance

Semana 7: Estrategias de Construcción de Software

Clean Code Frontend & Backend

¿Qué se realizó? Análisis de arquitecturas de desarrollo Frontend y Backend modernas. Implementamos buenas prácticas de codificación semántica (HTML5, CSS3, JavaScript limpio), modularización de componentes lógicos y flujos eficientes de control de versiones distribuidas utilizando Git.

Lo que entendí: El código no solo debe funcionar para la máquina, debe ser legible para los ingenieros que mantendrán el software. Entendí el valor estratégico del control de versiones distribuidas y las ramificaciones estructuradas en Git para trabajar en equipos de ingeniería eficientes.
Git Versioning Modularidad Web Standards

Semana 9: Tecnología Web Backend

Arquitectura Server-Side Servidores Web

¿Qué se realizó? Estudio de la arquitectura de aplicaciones web y el funcionamiento crítico del "server-side". Analizamos cómo los servidores web procesan peticiones, gestionan la concurrencia y se comunican con los lenguajes de backend para entregar contenido dinámico.

Lo que entendí: Comprendí la diferencia fundamental entre el procesamiento en el lado del cliente y el servidor. La arquitectura backend es el motor que garantiza la seguridad y la persistencia antes de que la información llegue al usuario final.
Server-side Backend Architecture

Semana 10: Aplicaciones Web con Python

Programación Backend Lógica de Servidor

¿Qué se realizó? Inmersión en el lenguaje Python aplicado al desarrollo web. Cubrimos desde la sintaxis básica, tipos de datos y estructuras de control hasta la programación orientada a objetos (POO) con clases, herencia, polimorfismo y manejo de excepciones.

Lo que entendí: Python es increíblemente versátil para el backend. Entendí cómo encapsular la lógica de negocio en clases ayuda a crear sistemas más ordenados y fáciles de depurar, facilitando la creación de APIs robustas.
Python POO Backend Logic

Semana 11: Introducción a Django

Patrón MTV Vistas y Modelos

¿Qué se realizó? Introducción al framework Django, explorando su arquitectura MTV (Modelo-Plantilla-Vista). Configuramos proyectos, gestionamos URLs, creamos vistas basadas en funciones y clases, y definimos modelos para la persistencia de datos.

Lo que entendí: La estructura de Django automatiza tareas pesadas. Entendí que el patrón MTV facilita la separación de responsabilidades, permitiendo que la base de datos y la interfaz se comuniquen eficientemente mediante las vistas.
Django MVC/MTV Web Framework

Semana 12: Django Admin, Sesiones y Auth

Autenticación Django Admin

¿Qué se realizó? Personalización avanzada de Django Admin y gestión de seguridad. Implementamos middlewares, manejo de sesiones de usuario, sistemas de autenticación y autorización para proteger las rutas críticas de la aplicación.

Lo que entendí: La seguridad no es opcional. Aprendí que gestionar sesiones correctamente y proteger el acceso mediante middleware es fundamental para mantener la integridad de los datos de los usuarios en entornos web.
Security Auth Middleware

Semana 13: Diseño de APIs RESTful

JSON / REST AJAX

¿Qué se realizó? Diseño e implementación de APIs siguiendo los principios RESTful y HATEOAS. Trabajamos en la serialización de datos complejos, filtrado de consultas, paginación y la protección contra vulnerabilidades como CSRF y CORS.

Lo que entendí: Las APIs son el puente de la web moderna. Aprendí que una buena API RESTful debe ser predecible, eficiente y, sobre todo, segura ante peticiones externas no autorizadas.
REST API Serialización CORS

Semana 14: Microservicios con Django

Microservicios Docker / K8s

¿Qué se realizó? Introducción a la arquitectura de microservicios. Construimos microservicios independientes con Django, explorando su comunicación, despliegue mediante Docker y Kubernetes, y estrategias para su monitoreo y escalado en producción.

Lo que entendí: Los microservicios permiten escalar partes específicas de una aplicación sin afectar el resto del sistema. Es un enfoque potente para organizaciones que buscan alta disponibilidad y agilidad técnica.
Microservices Docker Scalability
